# Rule: Double Documentation & Spanish Debrief
# Scope: Applies to all files and the terminal output

## Code Documentation
- **Internal:** Escribe comentarios en el código en **inglés** usando el formato JSDoc para funciones y constantes complejas. Explica el "porqué", no el "qué".
- **External:** Mantén un archivo `PROGRESS.md` en la raíz. Cada vez que completes una característica, añade una entrada breve con la fecha y lo que se implementó.

## Communication Protocol
- **Language:** Aunque el código y los comentarios internos sean en inglés, **todas** tus explicaciones en el chat y el resumen de tareas deben ser en **español**.
- **Post-Task Brief:** Al terminar una tarea, explica detalladamente qué archivos creaste o modificaste y por qué tomaste esa decisión arquitectónica.
